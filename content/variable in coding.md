#Coding 

A variable is an assigned value that you can manipulate. (holds information)

They can be any [[Datatype]]s, and in [[object oriented programming]], [[Game Object]]s are also included along with [[Datatype]]s because it's a shortcut. 

(you skip saying game.workspace.model.hibajaba.hibajabachild.hibajabagrandson), when you could just say hibajabachild.jibajabagrandson. AKA [[Referencing in Object Oriented Programming]].



# Why use variables?

## As a shortcut

You can think of variables as a storage box that can be opened when it has to be used.


![[Files/Images/variable in coding 2024-01-11 19.51.24.excalidraw\|600]]

![[Files/Images/variable in coding 2024-01-11 19.57.20.excalidraw]]

## As a way to test repeating values without having to change all of them
Sometimes in coding you have to test values just to see what *feels* right. Or if there is a bug in your code due to a value that is a part of your code a lot, then you want to be able to change it <mark class="hltr-yellow">quickly. You can do that easily without having to go through all the values, and instead using a variable and changing the value that way.
</mark>
- imagine it's a magic box that automatically changes the contents
- or theirs a factory that changes the boxes contents



## User input
To actually be able to use [[User Input]] you need to assign it to a variable, without the variable the value has nothing associated with it.

[[variable identifier]] = userinput

now you can use the userinput.
