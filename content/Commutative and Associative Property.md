# Associative Property
#math/arithmetic 

Same thing of [[Commutative and Associative Property]], just how you group them when doing [[addition]]


x+y+z = z+y+x

Doesn't matter the order with addition.


Take this example, if you have a block with height z, and a block with height y, and a block with length x, no matter what order you stack them in, it's going to have the same height if you stack all three, if you do x, y first, then z it's going to be the same as if you do z, y, then x.



# Commutative Property addition and multiplication
#math/arithmetic 


The Cumulative Property is basically saying that when you have [[addition]], it doesn't matter what order it's in, you just **do it**


y+x = x+y     it doesn't matter in what order you add things in because no matter what it equals the same thing


This is true even with subtraction, because [[subtraction]] is just addition with [[Negative Number]]


It's basically the same thing as associative propertyo just with getting rid of grouping is also valid.
