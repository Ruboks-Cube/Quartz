#Coding 

A class is often used in [[object oriented programming]]
- A class is like an object constructor, a blueprint for making objects
	- They’re like a “Type” → [[Datatype]]
	- you could create a class “person” and that person has different attributes and functions 
	- So whenever you create an object that is a “person” you can define those attributes and use those [[Method\|Methods]]
- A class can be thought of as a user-defined [[Datatype]]
	- They have properties (attributes) and methods
		- Properties are defined within the class and are used to define attributes or data of the class
		- Methods of a class affect their properties
