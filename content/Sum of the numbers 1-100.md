#math #problemsolving 

A simple [[addition]] problem that is a introduction to [[Problem Solving]]


What is:
1+2+3+4+...96+97+98+99+100

You want to solve this in an efficient way.
A quick [[Problem Solving]] tip, write down everything you know about the problem, and see how they relate.

- We know that there are numbers 1-100, not just 1-100, we also have 0, we can put 0 at the start, so now it's 0+1+2+3+4 ect
- We also know that there are an even amount of numbers inside the [[dataset]] (so not counting 0), this means that there is no "middle" since there is no middle, there's only 2 numbers in the middle.  1,2,3,4  see? no middle, but when you add 5, 3 becomes the middle, or [[Median]]

Another [[Problem Solving]] Tip:
Just try it in ways you didn't think about, first do the most obvious then try other things

1+2 is 3, 3+3 is 6, 6+4 is 10. This is getting us nowhere. But what is we do 1+100=101, 99+2 =101 also! and 98+3 = 101 too! So we have a pattern, each number has a pair to equal it to 101. How many pairs are there? There are a 100 numbers, so there are 50 pairs, and our answer is 101 * 50 which is 5050.


So you can see how we used to[[Commutative and Associative Property]] to rearrange the numbers in this problem so we can make it so each one has a pair.
